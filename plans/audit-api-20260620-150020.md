# Public API Audit - 2026-06-20 15:00 local

**Scope:** Manifest-public package subpaths from `public-packages.json` — 12 public packages,
94 public export subpaths, 1,789 enumerated exports.
**Lens:** Conservative JS framework API review (Next.js/TanStack/Vite); less public API is more,
compatibility is expensive, internals leak forever.
**Sources:** `SPEC.md`, `rules/api-surface.md`, `STABILITY.md`, `public-packages.json`,
`site/gen/api/*.md`, examples/site usage, source declarations.
**Commands:** `node .claude/skills/audit-public-api/scripts/inventory-public-api.mjs`
(1,789 exports across 94 subpaths) · `pnpm run check:api-surface`
(`public-exports-needing-attention=1367`, baseline=1367, fixed-this-run=0 — the gate passes
only because the entire surface is grandfathered into `api-surface-baseline.json`).
**Git:** branch `main`, commit `a817cb08`.
**Method:** Five parallel package-scoped sub-agents (core+style, server+better-auth,
browser+compiler+cli+create-kovo, drizzle+test, headless-ui+ui); each inspected source,
generated docs, example/site usage, and SPEC; main agent synthesized, normalized ranks, and
verified the four headline structural claims directly (see footnote).

## Executive Summary

- `Should keep`: ~120 symbols + the two large primitive surfaces (~34 headless-ui families,
  ~46 ui component families) — the framework's authoring core is well-grounded.
- `Borderline`: ~30 symbol families (mostly supporting-type bloat, undocumented-but-required
  types, and unproven/0-adoption surfaces that want `@experimental` or narrowing).
- `Definitely remove`: 11 symbol families (leaked internals, generated-ABI double-exports,
  and style-table exports STABILITY.md explicitly excludes from the versioned surface).

**Highest-leverage actions, in order:**

1. **Close the documentation gap — this is the #1 systemic finding, not a symbol problem.**
   `@kovojs/headless-ui` (~900 exports) and `@kovojs/ui` (~480 exports) have **no `apiRef`
   entry** and render into **zero** generated API reference; `@kovojs/server` ships **28
   undocumented public exports**; `@kovojs/test/html-fragment`'s **20** exports are all
   undocumented; the manifest declares a `browser-client` doc page that **does not exist**.
   Per STABILITY.md a symbol is "supported public" only if documented, so today's largest
   public surface is held together solely by the baseline ratchet.
2. **Remove the `@kovojs/ui` `*Styles` (46) and `*StyleOverrides` (36) exports** — STABILITY.md
   states style tables are *not* part of the versioned surface, yet they are versioned exports
   with zero example imports.
3. **Move generated ABI off human-public subpaths** — `@kovojs/browser/client` re-exports the
   `CompiledQuery*`/`QueryBinding*`/`Morph*` families that already live on `./generated`;
   `@kovojs/browser/client` is 4 real bootstrap helpers wrapped in 68 supporting types.
4. **Shrink the `@kovojs/server` `.` barrel (176 exports)** — drop the leaked internals
   (`PageHintOptions`, `StylesheetAsset`, `MutationSuccess`/`MutationResult` envelopes) and the
   dev-shell wiring already published on an internal subpath.
5. **Fix two concrete defects:** `@kovojs/drizzle` omits `KovoFanAnnotation` from its barrel
   though it is the element type of a public field (recursive-completeness violation); the
   inventory reads drizzle `.` from `runtime.ts` while the shipped barrel is `index.ts`.

## Definitely Remove

### `@kovojs/ui/*#*Styles` (46 `style.create(...)` tables)
**Current exposure:** Every `@kovojs/ui` subpath exports its compiled style table as a named
public export (e.g. `packages/ui/src/accordion.tsx:62` `export const accordionStyles`), plus side
variants (`drawerSideStyles`, `sheetSideStyles`).
**Evidence:** STABILITY.md L27-28: "Compiled atomic-class strings and **internal style tables are
not part of the versioned surface**." Zero example imports of any `*Styles` table; customization
goes through the `styles={...}` prop. Undocumented (no JSDoc → violates rules/api-surface.md).
**Assessment:** Exporting these as versioned symbols invites apps to depend on atomic-class
identity the policy explicitly reserves the right to change every release. ~46 dead public symbols.
**Recommendation:** Stop exporting `*Styles`/`*SideStyles`; keep them module-private (render
functions already close over them). For the copy-in starter the table is part of the copied file.
**Migration note:** Zero importers — no external migration.

### `@kovojs/ui/*#*StyleOverrides` (36 interfaces)
**Current exposure:** 36 `*StyleOverrides` interfaces (e.g. `accordion.tsx:19`), each a
`Record<part, style.StyleInput>` typing the `styles={}` prop, naming internal per-part slot keys.
**Evidence:** STABILITY.md L27-28 (the override vocabulary is keyed to the non-versioned style
tables); zero example references; reachable structurally via `*Props['styles']` so a *named*
export is unnecessary.
**Assessment:** Naming 36 distinct override interfaces freezes the internal part decomposition as
a versioned vocabulary, for a capability callers reach by passing an object literal.
**Recommendation:** Inline the override shape into `*Props['styles']` (anonymous), or expose one
documented generic `StyleOverrides<Part>` helper. Do not export 36 interfaces.
**Migration note:** Structural typing still checks object literals; no named import to migrate.

### `@kovojs/server/.#PageHintOptions`
**Current exposure:** Root barrel via `api/rendering.ts`; `hints.ts:61` bag of `bootstrapScript`,
`modulepreloads`, `prerenderUrls`, `i18n`, `meta`, `prefetch`.
**Evidence:** Undocumented (server.md "*Undocumented.*"); never a parameter/return of any
documented public function — consumed only by internal render-shell code (`document-core.ts`,
`vite-manifest.ts`, `route.ts`); 0 example imports. SPEC §9.5 L918 makes page assembly
framework-owned.
**Assessment:** Internal render-shell hint bag gratuitously published; leaks page-assembly
vocabulary the SPEC reserves to the shell.
**Recommendation:** Drop from the barrel; move to `@kovojs/server/internal/*`.
**Migration note:** No public consumer.

### `@kovojs/server/.#StylesheetAsset`
**Current exposure:** `hints.ts:30` (`criticalCss`/`cspHash`/`href`/`preload`); reachable because
`stylesheet()` returns it and `createApp({ stylesheets })` accepts `string | StylesheetAsset`.
**Evidence:** Undocumented; `cspHash`/`criticalCss` are compiler/CSP-pipeline outputs, not author
inputs. The author input type is the documented `StylesheetDeclarationOptions`. Examples call
`stylesheet('/app.css', {...})` and never name `StylesheetAsset`.
**Assessment:** A leaked internal asset record standing in public signatures where an opaque
handle belongs; exposes CSP/critical-CSS pipeline internals.
**Recommendation:** Make `stylesheet()` return an opaque/branded handle; keep
`StylesheetDeclarationOptions` as the only author-named type.
**Migration note:** Authors pass strings or `stylesheet(...)` results; the named type rarely
appears in app code.

### `@kovojs/server/.#{MutationSuccess, MutationResult, MutationFail}` (result envelopes)
**Current exposure:** `mutation.ts:80/90/100` via `api/data.js`. `MutationSuccess` exposes
`changes`, `rerunQueryInstances?`, `rerunQueries`, `responseHeaders` — the internal result envelope.
**Evidence:** All undocumented; 0 direct example imports (the inventory's `total=24` is
package-internal). SPEC §6.3 shows authors return plain values or `fail('CODE', payload)` and
read typed failures via `forms.<mutation>.failure` — they never construct an envelope.
**Assessment:** Internal wire/result envelopes leaked as public, exposing query-rerun machinery.
**Recommendation:** Move `MutationSuccess`/`MutationResult` internal; if `MutationFail` must stay
(used by `AppMutationResponseOptions.renderFailureFragment`) document it and narrow to the
author-visible `{ code, payload }` shape.
**Migration note:** Authors use `fail()` and `forms.*.failure`.

### `@kovojs/server/.#{createKovoAppShellViteDevIntegration, kovoAppShellViteDevPlugin}` (remove from `.` barrel)
**Current exposure:** Root barrel (`index.ts:18`, `@experimental`), **and** already published on
`@kovojs/server/internal/app-shell-vite`.
**Evidence:** 0 example/site imports; SPEC §9.5.1 L931-936 says authors should *not* hand-wire
the dev refresh registry — the public `kovo()` plugin (from `./vite`) owns it and calls these via
the internal subpath (`vite.ts:171`).
**Assessment:** Duplicate dev-shell wiring on the public barrel for something the SPEC steers
authors away from touching.
**Recommendation:** Remove from the root `.` barrel; keep only on the internal subpath.
**Migration note:** Authors use `kovo()` from `@kovojs/server/vite`.

### `@kovojs/style/.#{defineVars, createTheme, Vars, Theme}` (StyleX ABI)
**Current exposure:** Style root barrel (`index.ts:1-14`).
**Evidence:** Only consumer is the compiler (`packages/compiler/src/style.ts:6-7,87-103`); 0
example/site usage. SPEC §13.1 names these only as compiler-*extracted* forms; apps theme via
`defineTheme`/`tokens`. The opaque return types `Vars`/`Theme` surface private markers
(`CSS_MARKER`, `STYLE_SRC`, `__rules`, `__vars`, `AtomicRule`) into the public `.d.ts`.
**Assessment:** The raw StyleX fork point, redundant with the documented `defineTheme` seed path,
leaking compiler-internal shapes.
**Recommendation:** Move to `@kovojs/style/internal` (compiler imports the package directly
regardless of subpath); if kept public for advanced manual token groups, mark `experimental_`.
**Migration note:** No example or `@kovojs/ui` source calls them — compiler-only.

### `@kovojs/style/.#InlineStyle`
**Current exposure:** Style root type (`index.ts:6`).
**Evidence:** Its own generated doc (`style.md:120-125`): "used only through the **internal**
`raw(...)` escape hatch." `raw` is not a public export; 0 usage.
**Assessment:** A public type that only types an internal function — dead public surface, alive
only because `StyleInput` references it.
**Recommendation:** Fold the raw tuple form into `StyleInput`'s definition and move both internal.
**Migration note:** No consumer imports it by name.

### `@kovojs/core/.#{formFields, FormFieldName}`
**Current exposure:** `index.ts:638-721`.
**Evidence:** 0 example/site/`@kovojs/ui` usage (the lone `formFields*` example hit is an

**Recommendation:** Remove `formFields` (move to starter/example); demote `FormFieldName` unless

### `@kovojs/browser/client#{CompiledQuery*, QueryBinding*, ImportHandlerModule}` (generated-ABI double-export)
**Current exposure:** `type`-only re-export from public `./client` (`client.ts:81-89`) — **also**
exported from `./generated` (`generated.ts:8,14-26`), the subpath the compiler emits against
(`compiler/src/emit/client.ts:22`).
**Evidence:** All carry generic generated-ABI JSDoc; reach `./client` only because
`KovoLoaderOptions.queryPlans?: CompiledQueryUpdatePlans` drags them in via recursive publicness —
but `queryPlans` is how *generated bootstrap code* feeds plans to the loader, not human config.
0 app-source usage. SPEC §5.2#8: `@kovojs/*/generated` is compiler-emitted ABI, not app API.
**Assessment:** Pure generated-ABI leaking onto a human-public subpath; already correctly on
`./generated`.
**Recommendation:** Remove from `./client`; retype `installKovoLoader`'s `queryPlans` against a
small opaque public type (or accept the generated type only on the `./generated` re-export).
**Migration note:** Consumers needing them are generated code already importing `./generated`.

### `@kovojs/browser/client#{Morph*, StructuralMorph*, TargetCollectorRoot}`

**Recommendation:** Move to `./generated` or `./internal/morph`; re-point the test helper.
**Migration note:** One import in `app-test-helpers.ts`.

## Borderline

### `@kovojs/core/.#WebhookVerifier kit` (19 symbols)
**Current exposure:** 3 functions (`customVerifier`, `hmacSignature`, `standardWebhooks`) + 16
types on the core root.
**Evidence:** SPEC §9.1 L862 makes the verifier kit normative for `webhook()`, **but** 0
`webhook(` call sites in examples. The constructors + their direct input types are SPEC-mandated;
the derived/resolved types (`ResolvedHmacSignatureConfig`, `HmacSignaturePayloadContext`,
`HmacMultiSignature`, `HmacSignatureEncoding`, `HmacSignatureVerifier` internals) are config
internals with zero usage.
**Assessment:** SPEC-grounded but over-broad relative to the mandate, and the whole kit ships with
no example exercising `webhook()` — the contract is unproven.
**Recommendation:** Keep the 3 constructors + direct input types + `WebhookVerifier`; `@internal`
or fold the derived/resolved types into the option objects. Add a webhook example before freezing.

### `@kovojs/core/.#Storage capability family` (7 types)
**Current exposure:** `StorageCapability` + 6 result/option types (`index.ts:12-20`).
**Evidence:** `@kovojs/server/src/schema.ts:1,192` types `storage: StorageCapability` on a server
context (real cross-package contract), **but** no SPEC §10 reference to object storage and 0
example usage.
**Assessment:** A genuine cross-package interface that is nonetheless unproven and SPEC-silent;
lives in core only to be re-typed by server.
**Recommendation:** Relocate the family to `@kovojs/server` (its only consumer), or gate behind
`experimental_` until SPEC sanctions object storage and an example exercises it.

### `@kovojs/style/.#Theme generator type family` (17 types incl. `KovoTheme`)
**Current exposure:** `theme.ts` — `defineTheme` returns the whole `KovoTheme` (`css`, `ref`,
`light`/`dark`, `sys`, `custom`, `variant`).
**Evidence:** Examples only read `style.tokens.sys.color.*` (typed `var()` refs), never the
concrete generated values. The `Theme*Values`/`ThemeReferencePalettes`/`ThemeSchemeValues` types
are the Material engine's *output internals* surfaced because the return is the whole object.
**Assessment:** Returning the full generator object couples the public theme contract to the
Material implementation's shape.
**Recommendation:** Narrow `defineTheme`'s return to `{ css }` + `tokens`; keep the *name* unions
(`ThemeSystemColorName`, `ThemeVariant`) and input options public; demote concrete-value types.

### `@kovojs/core/.#supporting type tails` (Query bindings, GET-form, form-failure)
**Current exposure:** `query`/`Query`/`QueryArgsBinding`/`QueryRefreshBinding`/`QueryRefreshSpec`;
`GetForm`/`GetFormDescriptor`/`GetFormInput`; `ComponentMutationFormState`/`FormValidationFailure`.
**Evidence:** All SPEC-grounded (§4.9/§10.2 queries, §6.3 GET forms, §6.3/§9.2 failures) and
structurally required by other public types, but 0 direct example imports.
**Assessment:** Recursively required → must stay public, but unexercised; the fluent builder types
and descriptors could be narrowed/inlined into their parent interfaces.
**Recommendation:** Keep; add proving examples (search forms, component query bindings); inline
the builder/descriptor types into parent signatures where convenient.

### `@kovojs/server` undocumented author-facing type families
**Current exposure:** Guards callbacks (`UnauthenticatedContext`, `UnauthenticatedHandler`,
`ForbiddenContext`, `ForbiddenRenderer`); webhook context family (`WebhookHandlerContext`,
`WebhookChangeOptions`, `WebhookFail`, `WebhookTransactionContext`, `WebhookDefinition`,
`WebhookDeclaration`); mutation author types (`MutationDefinition`, `MutationContext`,
`MutationFormDefinition`); validation/meta supporting types (`ValidationIssue`,
`ValidationFailurePayload`, `I18nCatalog`, `RouteMeta`, `RouteMetaFactory`, `RouteMetaSource`,
`RoutePrefetch`); `CookieOptions`; `StaticExportCompileDiagnostic`.
**Evidence:** All undocumented (server.md "*Undocumented.*") yet reachable from documented public
functions (`createApp`, `webhook()`, `mutation()`, `meta()`/`metaFromQuery()`) — so per
rules/api-surface.md they must be public **and** documented. `MutationDefinition` is additionally a
cross-package contract (better-auth returns it).
**Assessment:** Rule violations of the "every referenced helper type must be documented public"
clause, not removal candidates — except `RouteMetaFactory`/`RouteMetaSource` (overload glue,
inlinable) and the guards callbacks (possibly redundant with `ErrorShellRenderer`).
**Recommendation:** Document all with SPEC cites (§6.4/§6.5/§6.6/§8/§9.5); inline the meta-overload
glue; confirm the guards callbacks are reachable from a live `createApp` option or demote them.

### `@kovojs/server/.#unproven verbs & wire vocabulary`
**Current exposure:** `write`/`WriteDefinition` (0 adoption); `tag` (alias of `domain`); `stream`
+ the 8-type `MutationStream*` family (documented, 0 example use); typed-nav supporting-type zoo
(`LinkDescriptor`, `Redirect`, `RouteResponse*`, header-bag aliases `MutationResponseHeaders` =
`ResponseHeaders`); `exportStaticApp`/`StaticExport*`; `MutationReplayStore`/
`createMemoryMutationReplayStore`.
**Evidence:** SPEC-grounded (§8 typed nav, §9.1 streaming, §9.5 static export/replay) but
0-adoption in example/site apps; `tag`=`domain` and the header-bag aliases are pure vocabulary
duplication.
**Assessment:** Real features but heavy/unproven for a v1 contract.
**Recommendation:** Keep the SPEC-mandated verbs (`href`/`Link`/`redirect`/`stream`); mark
`write`/`tag`/streaming `@experimental` until a corpus example proves them; collapse the
header-bag alias pairs to one type; move `createMemoryMutationReplayStore` to starter code (the
`MutationReplayStore` injection type stays).

### `@kovojs/browser/client#supporting-type bloat` (~50 `*Like`/event/observer/upload types)
**Current exposure:** `type`-only re-exports forming the recursive option surface of the 4
bootstrap helpers (`client.ts:20-100`).
**Evidence:** 0 example/site usage; present only as transitive option-type dependencies; ~41 carry

**Assessment:** Defensible under the letter of recursive publicness (so the gate accepts them) but

**Recommendation:** Narrow the option types (`KovoLoaderOptions`, `CreateBrowserKovoRootOptions`,
`EnhancedMutationLoaderOptions`) to a minimal named set; demote the structural `*Like` interfaces

### `@kovojs/cli/.#runKovoCommand`

### `@kovojs/test` surface

### `@kovojs/drizzle/.#KovoFanAnnotation` (reachability defect)

### `@kovojs/headless-ui#type tails & near-duplicates`
**Current exposure:** Toast's 12-symbol event-definition kit (`toastEvents`,
`ToastEventDefinition`, `toast*Event`/`*EventName`/`*Payload`); per-primitive intermediate

**Assessment:** Families are structurally sound; the type sprawl and toast event kit are the
borderline parts; the disclosure/menu duplication is a product question.
**Recommendation:** Audit *type* exports per family — collapse `*ChangeDetail/Reason/Result` into a

## Cross-Cutting Recommendations

1. **Documentation is the dominant gap and should be the first workstream.** The largest public
   surface in the framework (`@kovojs/headless-ui` ~900 + `@kovojs/ui` ~480 exports) has no
   `apiRef` and renders into zero reference; `@kovojs/server` has 28 undocumented exports;
   `@kovojs/test/html-fragment` has 20; the manifest declares a `browser-client` doc page that
   does not exist on disk. Per STABILITY.md these are not "supported public" until documented, yet
   `pnpm run check:api-surface` passes only via the 1,367-entry baseline ratchet. Either add
   `apiRef` + JSDoc for these surfaces or mark the unfrozen tail `@experimental` — today policy
   and surface disagree.
2. **Generated ABI must not sit on human-public subpaths.** `@kovojs/browser/client` is the worst
   offender (CompiledQuery*/QueryBinding*/Morph* already on `./generated`); the same hygiene check
   should run against every public subpath, keyed off the generic generated-ABI JSDoc string.
3. **No public signature should name an internal/generated type.** Confirmed leaks:
   `@kovojs/test/harness` `touchGraph?: CoreGraph.TouchGraph` (core *internal* subpath);
   `@kovojs/style` opaque types expose private markers in their `.d.ts`; `@kovojs/server`
   `StylesheetAsset`/Mutation envelopes; `@kovojs/better-auth` returns the *undocumented*
   server `MutationDefinition`. Promote the supporting type, brand it opaque, or move the symbol
   behind an internal subpath.
4. **Encode `@kovojs/ui`'s dual distribution in the manifest.** It is classified `library` but
   STABILITY.md treats it as library + `starter` copy-in; the versioned-vs-copied boundary
   (factories/Props versioned; style tables not) is what makes the `*Styles` removal safe, so the
   `kind`/manifest should record it.
5. **Right-size the v1 contract.** The `@kovojs/server` `.` barrel (176) and `@kovojs/browser/
   client` (72) are too large for a clean v1; prefer fewer, documented, example-proven exports and
   `@experimental` on unproven surface (streaming mutations, `write`/`tag`, static export).
6. **Fix the drizzle inventory/source mismatch.** The inventory enumerates `.` from `runtime.ts`
   (8 exports incl. `KovoFanAnnotation`) while the shipped barrel is `index.ts` (7, without it);
   reconcile so the audit baseline and the published surface agree.

## Gaps And Follow-Up

- **The api-surface gate is currently a ratchet, not a clean gate.** `check:api-surface` reports
  1,367 public-exports-needing-attention held at baseline; it does not certify the surface is
  documented/clean, only that it has not grown. Headline doc claims above were verified directly
  against source and `site/gen/api/`, not via the gate.
- **The `kovo add` copy-in flow has zero example/test exercise** (no vendored components anywhere
  in `examples/`). The premise that justifies co-locating style tables for the starter is
  unverified; if copy-in is a real v1 distribution it needs a fixture, otherwise the `*Styles`
  exports cannot be defended as "needed for the starter."
- **headless-ui per-symbol "0 usage" is noisy.** `examples/gallery/src/interactive-gallery-
  harness.ts` does `import * as Primitives` across all 34 components, so usage markers over-count
  bulk-imported handlers; judge headless-ui by family, not per-symbol counts.
- **Product decisions deferred:** whether `disclosure`/`collapsible` and the four menu primitives
  should remain distinct versioned subpaths; whether object storage (`StorageCapability`) and
  streaming mutations belong in v1 or `@experimental`.
