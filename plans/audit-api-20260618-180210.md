### Highest-leverage removals / narrowings

1. **`@kovojs/ui` is misclassified as a versioned public library.** STABILITY.md, `packages/ui/registry.json`, `copy-in.test.ts`, and SPEC's vendored-source model all describe it as a shadcn-style **copy-in starter** ("you own the code"), yet the manifest publishes it `visibility:public / kind:library` with 44 subpaths and **1,479 exports — 845 of them `@kovojs/headless-ui` primitives re-exported via banned `export *`**. This single misclassification is the entire reason `check:api-surface` fails (872 undocumented exports). **Reclassify `kind:starter` + `private`; drop the `export *` re-exports.**
2. **`@kovojs/runtime/client` is ~85% loader internals on a human-public subpath.** ~95 of 115 exports (morph engine, query-binding apply, enhanced-mutation submit, optimistic rebaser, broadcast, refetch, event bus, submit-context, inline-query hydration) have **zero non-test usage**; the only real app entry (`create-kovo` starter) imports 7 symbols. Shrink `./client` to ~`installKovoLoader` + `KovoLoader(Options)` + `createQueryStore` + a few morph/target types; move the rest behind the already-declared `./internal/*` subpaths.
3. **`@kovojs/server` root barrel leaks request-shell + wire internals** — ~37 undocumented types (document-assembly, deferred-stream, query-endpoint wire, route-page result, diagnostic-document) whose consuming functions already live **only** on `./internal/{html,wire,route}`. Relocating them also fixes recursive-publicness leaks (`RoutePageFailure`/`QueryEndpointFailure` name the `@internal` `ResolvedGuardFailure`).
4. **`@kovojs/test` over-publishes 13 subpaths; ~61 of 83 exports are undocumented.** `harness-operations`, `sql-observer`, `verifier-sql`, `verifier-diagnostics` (minus `DbVerificationDiagnostic`), and the unused 2/3 of `html-fragment` are verifier/instrumentation internals consumed only by framework tests + the **private** conformance package. Demote to internal subpaths.
5. **Generated ABI is sitting on human-public/undocumented subpaths.** `@kovojs/server/jsx-runtime` + `jsx-dev-runtime` (16 symbols, the JSX automatic-runtime ABI) and `@kovojs/core`'s `FragmentTargets`/`ComponentRegistry`/`LiveTargetRegistry` seeds should move to `apiBoundary.generated` + `@generated`, mirroring `@kovojs/runtime/generated`.
6. **Manifest gaps:** `@kovojs/server/./vite` is published but **unclassified** (and it is the SPEC §9.5 dev entry); `@kovojs/server/app-shell/{core,node,client-modules}` are public subpaths that export **nothing** (`export {}`); `@kovojs/test` `.` is an empty documented entry.
7. **Recursive-publicness violations in 5 packages** — public signatures that cannot be named without importing internal/unexported types: `@kovojs/drizzle/derive` (built entirely from `@kovojs/core/internal/derivation`), `@kovojs/cli` `kovoCheck`/`kovoExplain` (name `@kovojs/core/internal/graph` `@internal` types), `@kovojs/test` `sql-observer`/`verifier` (`ObservationRecorder` / `TouchGraph`), and many `@kovojs/runtime/client` option types (`QueryApplyInterposition`, `OnDeltaMiss`, `RuntimeErrorReporter`, `FragmentChunk`).

#### @kovojs/ui — I-ui

### `@kovojs/ui/* — 32 component barrels re-export the entire @kovojs/headless-ui primitive surface via `export *`` — covers 845 symbol(s) _(skeptic re-ranked → **borderline**)_

**Current exposure:** Each component file ends with `export * from '@kovojs/headless-ui/<x>';` (e.g. tooltip.tsx:183, popover.tsx:188, command.tsx:546). 845 of the 1479 slice symbols are declared in packages/headless-ui/src and re-exported through ui subpaths — every primitive's *Attributes, *State, *Options, *Event, *ChangeReason/Detail/Result, set*/toggle*, *KeyDown/*Click handlers, etc.

**Evidence:** grep 'export \* from \'@kovojs/headless-ui' across packages/ui/src = 32 files; slice table per-subpath origin breakdown sums to 845 headless-declared symbols (e.g. command 55, navigation-menu 47, autocomplete 44, toast 44). VIOLATES rules/api-surface.md:67-70 'No export * on a public barrel' (auto-publishes every current and future symbol, silently widening surface). It also double-publishes the @kovojs/headless-ui surface (slice context: 'each ui/<x> barrel re-exports the entire headless primitive surface'). This is why `pnpm run check:api-surface` currently FAILS — hundreds are undocumented (slice context cites @kovojs/ui/tooltip#tooltipTriggerPointerEnter).

**Assessment:** This is the headless re-export leak. A copy-in starter should never re-publish its versioned dependency's full surface — apps that need primitive behavior import @kovojs/headless-ui directly (the versioned dep per STABILITY.md:23). The `export *` mechanism is doubly wrong: it is forbidden on public barrels by api-surface.md, AND it widens the headless surface into ui where it is undocumented. The ONE legitimate reason these resolve through ui today is the compiler: generated client modules import primitive handlers via the ui subpath (examples/gallery/src/generated/interactive/popover-demo.client.js:4 imports popoverBeforeToggle from '@kovojs/ui/popover'). That is a generated-ABI resolution concern, not evidence of human-public demand — and it should resolve the handler through @kovojs/headless-ui or a generated subpath, not a human-public ui barrel.

**Recommendation:** Drop every `export * from '@kovojs/headless-ui/*'` from ui component files. Component .tsx files should keep importing the specific headless symbols they use (named imports, as tooltip.tsx:4-7 already does) but NOT re-export them. Fix the compiler's generated-client handler resolution to target @kovojs/headless-ui (or a generated subpath) directly so generated modules stop depending on the ui re-export.

**Migration note:** Apps needing primitive behavior import @kovojs/headless-ui/<x> directly. Update compiler handler-import emission before removing the re-exports so generated client modules keep resolving.

**Skeptic verification:** Downgrade remove->borderline because the `export *` is currently a LOAD-BEARING generated-ABI contract: emitted .client.js modules resolve primitive handlers through the ui re-export, so a bare deletion breaks already-emitted/regenerated client modules until the compiler's handler resolution is repointed at @kovojs/headless-ui. This is genuinely strong grounding for sequencing, not for keeping the surface: the re-export still VIOLATES rules/api-surface.md:67-70 ('No export * on a public barrel'), double-publishes the entire headless surface, and is the documented cause of the failing check:api-surface gate (e.g. undocumented @kovojs/ui/tooltip#tooltipTriggerPointerEnter). ## Cross-Cutting Recommendations

1. **Adopt a `generated` tier consistently.** Four packages put compiler-emitted ABI on a human-public barrel: `@kovojs/core` registry seeds, `@kovojs/server` jsx-runtime, `@kovojs/drizzle/derive`, and `@kovojs/style` (`createAtomicStyles`/`AtomicCssResult`). `@kovojs/runtime/generated` is the model to mirror — published + typed for the compiler, excluded from the human API reference, `@generated`-tagged. `@kovojs/style` and `@kovojs/runtime` already declare `./internal` subpaths that are the natural home.
2. **The documentation gate is currently aspirational.** `@kovojs/headless-ui` (~930/934 undocumented **and no `apiRef` page at all**, despite being the package STABILITY.md singles out as the versioned contract), `@kovojs/test` (61/83 undocumented), and `@kovojs/server` (~115/270) carry large undocumented public surfaces. The api-surface ratchet is being satisfied by boilerplate JSDoc that does not catch dangling unexported-type references (notably across `@kovojs/runtime/client`). Either document or demote before v1, and tighten the gate to detect non-public types reachable from public signatures.
3. **Eliminate `export *` on public barrels.** `@kovojs/ui` uses `export *` from headless-ui in 32 files (hard violation + the cause of the gate failure). `@kovojs/server`'s root `index.ts` uses `export *` from curated sub-barrels (`api/{data,rendering,routing}`) — softer because those sub-barrels are explicit, but the literal rule targets the public entry; convert to explicit named re-exports.
4. **Resolve "two of everything" before freezing v1.** `@kovojs/style`: two token systems (`defineTheme`/`tokens` vs `defineVars`/`createTheme`) and two merge APIs (`attrs` vs `props`). `@kovojs/server`: `kovo()` vs `createKovoViteIntegration()`, and root `KovoAppShell*` dev plugins vs the `./vite` entry. `DiagnosticCode` is exposed from four entries (core, cli, test/verifier-diagnostics, drizzle) — keep it on `@kovojs/core` only.
5. **Use `@experimental` for real-but-unproven capabilities** rather than freezing them: storage adapters + S3 shim (core), `event()` family (core), webhook replay store (server), deploy presets (server/build), better-auth `mount`/sign-up, and the entire headless-ui reducer/handler layer (which has no documented hand-authoring story and zero non-gallery usage). Marking them now keeps v1's SemVer surface honest.
6. **Sequence the headless re-export removal carefully.** The only real consumers of `@kovojs/ui`'s headless re-exports are **compiler-generated** client modules importing primitive handlers (e.g. `popoverBeforeToggle`) via `@kovojs/ui/<x>`. Retarget the compiler's handler-import emission to `@kovojs/headless-ui` (or a generated subpath) **before** dropping the ui `export *`, or generated client modules break.

## Gaps And Follow-Up

- **`pnpm run check:api-surface` fails today** (872 undocumented `@kovojs/ui` exports). The audit treated this as a finding, not a blocker; the gate cannot pass until `@kovojs/ui` is reclassified/private or every re-exported symbol is documented. This is the most actionable single fix (Finding 1).
- **Generated-docs drift:** `site/gen/api/style.md` documents a public `CreateOptions` type and a `create(styles, options)` call shape that **do not exist in source** (source declares the unexported `StyleIdentityOptions` and uses overloads). Regenerate/reconcile the reference, or export `StyleIdentityOptions` as `CreateOptions` so docs and source agree.
- **No generated API reference for `@kovojs/headless-ui`** — a documentation gap for the one versioned-contract package, not an API-shape finding.
- **Product-owner judgment required on:** (a) whether `@kovojs/ui` ships as an installable package at all vs. pure copy-in; (b) whether `kovoCheck`/`kovoExplain` should freeze a public verifier-graph contract or accept an opaque runtime-validated input so app code never names the IR; (c) whether the headless-ui low-level reducer/handler layer is an intended hand-authoring surface or should become compiler-only/generated.
- **Method limits:** audit-only — no source edits, and no build/test beyond `check:api-surface`. Coverage counts are agent-reported reviewed exports; the slice C2/E notes record minor family-boundary double-counts (a couple of symbols counted in two adjacent families) that do not change any rank. `@kovojs/better-auth`'s 13 public symbols pass the gate, but their full structural types are nameable only via the explicitly-unstable `./internal` subpath — flagged as a deliberate-curation question, not a violation.
