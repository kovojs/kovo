## HIGH

  - **Verified:** Round-1 worktree vitest drove the real `extractQueryFactsFromProject`/`diagnosticsForQueryFacts`/`scopeAuditsFromQueryFacts`; literal `query(...)` → fact + KV435 + `scope:'args'` audit; aliased and namespace forms → `facts === []`, no KV435, `scopeAudits === []`.   - **Distinct:** new primitive (`query`) and new gate family vs `bugz-2` H2 (`kovo()`)/H3 (`route()`) and `bugz` H4 (`sql`); not the `bugz` H3/H5 predicate-operand name laundering (there the query is recognized; here it never is).
  - **Fix:** resolve `query` to its `@kovojs/server` import binding (aliases + `<ns>.query`) by AST symbol-identity at all three sites, mirroring the `kovo()`/`route()` hardening. Optionally reconcile the runtime query registry against the static fact set and fail closed on any served-but-unrecognized query.

## Refuted / not carried forward

- **Framework signing keyring accepts a 1-byte secret (no entropy floor)** — refuted: `createApp` → `validateAppEnv` enforces a fatal 32-char minimum (and low-entropy warning) in production at the single bootstrap chokepoint; all three HMAC families (CSRF, capability-URL, live-target) consume that one validated secret. A sub-floor secret is reachable only by hand-rolling crypto outside `createApp`.
- **`stream-text` data-stream-renderer dynamic import skips the allowlist (cross-origin module import)** — refuted as reachable: `StreamTextBuffer` is dead/unwired — never imported by `installKovoLoader`, never constructed in production, and not in the `@kovojs/browser` exports map; the shipped inline-loader streaming path routes through the `ki`-guarded importer. Latent only — harden the assert if `StreamTextBuffer` is ever wired.
- **CSP report aggregate LRU eviction is weaponizable (defender blinding)** — refuted: bounded eviction is by-design memory safety (the only alternative is unbounded-growth DoS); the snapshot has no shipped reader (write-only dead storage), and eviction under flood is inherent to any bounded attacker-writable telemetry buffer.
- **Stored CSP report fields are not HTML-escaped (latent stored XSS)** — refuted: escaping is correctly contextual/render-time (storage escaping would corrupt JSON/log consumers); no shipped renderer/sink exists; out-of-threat-model (the universal "app renders raw string" footgun).
- **`compareAndSet` mis-reads a Drizzle `.returning()` array as a CAS conflict** — refuted: `DrizzleUpdateResult` is a weak type, so pairing CAS with `.returning()` is a hard TS2345 compile error caught by `kovo check` (tsc); reachable only via `as any`. Also fails safe (spurious 409 + rollback).
- **Endpoints lack a scoped rate limit / per-principal `rateLimit` guard (looser DoS budget)** — refuted: SPEC-conformant by design — endpoints DO receive the mandated coarse per-IP+global floor; the tighter `/_m//_q/` rates and the per-session guard chain are intentionally inapplicable to the session-free `appOwnedSafety` escape hatch (a 120/min cap would break webhooks/LLM-proxies).
