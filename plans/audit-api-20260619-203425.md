## Executive Summary

- **`Should keep`:** ~9 explicit families + the documented keep-sets in the coverage ledger
  (≈530 symbols across the authoring cores of `core`, `server`, `browser`, `style`, `cli`, `drizzle`,
  `test`, `better-auth`, plus the `headless-ui` `*Attributes` builders and `ui` Component/Props).
- **`Borderline`:** **22 families** (19 originally ranked borderline + 3 reclassified down from
  `remove` by the adversarial pass).
- **`Definitely remove`:** **4 families, ~370 symbols** — and only these four survived verification
  (4 of 7 proposed removals confirmed; 2 overturned, 1 softened).

**What changed since the 2026-06-18 audit (`f8351a0f` → today).** Three prior recommendations landed:
(1) `@kovojs/runtime` was renamed **`@kovojs/browser`** (`./client` + `./generated` moved with it);
(2) the **`@kovojs/ui` `export *` violation is gone** (0 wildcard re-exports remain — the 845-symbol
re-export that failed the gate was deleted; `ui` now imports headless symbols by name per component);
(3) a **`generated` tier exists** for `@kovojs/browser` and `@kovojs/server` JSX. The `check:api-surface`
gate no longer _fails_ — it was converted to a **baseline ratchet of 1,571 untagged/undocumented public
exports** that shrinks as `plans/api-cleanup.md` Phases 4-8 land. **96% of that 1,571 is the two
primitive packages** (`headless-ui` ~900, `ui` ~600). The doc/type debt is tracked, not paid.

**The adversarial pass is the headline.** The package auditors, using a "0 named-imports / 0
example-imports" heuristic, proposed removing **~430 headless-ui machinery types and the test-harness
option types**. The skeptics **overturned** these: a return/param type is load-bearing in a public
contract even when no consumer ever _imports its name_, because TypeScript infers it from the kept
function's signature. Demoting them to `@internal` would make a kept public function's return type
un-nameable from any public subpath — a fresh recursive-publicness violation in the other direction.
The lesson for this codebase: **"unused by name" ≠ "removable" for inferred types**; the gate's flat
baseline hides which of the 1,571 are genuinely dead vs. forced-public.

**Highest-leverage actions, in order:**

1. **Reconcile `@kovojs/ui` to dual distribution (owner decision 2026-06-19).** `@kovojs/ui` is **both**
   a versioned library **and** a shadcn-style copy-in starter — the two sources are each wrong by
   describing only one mode: `STABILITY.md` says copy-in only, while `public-packages.json` +
   `package.json` (`kind:library`, no `private`, 45 `publishConfig` subpaths at `0.1.1`) say library
   only. **Fix both to describe both:** keep `kind:library` + published + the `registry.json`/`kovo add`
   copy-in path; rewrite the `STABILITY.md` paragraph to "versioned library and copy-in." The library
   half then owes the discipline below (document/type the surface, un-export the class strings, get an
   api-ref page); the 240 compiled-class strings are un-exported regardless (they belong to neither
   contract). This supersedes `plans/api-cleanup.md` Locked Decision #2 / Phase 7 (which set `ui`
   `private:true`, copy-in only).
2. **Un-export the 240 `@kovojs/ui` compiled-class strings** (`*Styles`/`*Classes`/`*ClassNames`).
   Adversarially confirmed: zero external consumers, pure StyleX build output. **Clears 240 of the 1,571
   baseline violations with no capability loss** — the single biggest immediate win.
3. **Re-tier the headless primitive ABI — but this is GATED, not immediate.** The 124 reducers
   (`set*`/`toggle*`/…) and the 100 `@kovoPrimitiveHandler` functions are confirmed not-app-facing, but
   `plans/api-cleanup-leftover.md` (closed 2026-06-19) records a standing decision to **keep them public**
   until a **generated handler ABI + a different L1 island-authoring story** land: emitted client modules
   hand-import the handlers (so they can't be `internal` — `rules/api-surface.md` bars emitted code from
   `internal`), and the gallery hand-imports the reducers. So the handlers move to a new `generated` tier
   only once that tier exists, and the reducers move to `internal` only once the L1 authoring path stops
   hand-importing them. Sequenced behind that prerequisite, not a free baseline drop.
4. **Shrink `@kovojs/browser/client` from 72 → ~8 documented symbols.** 64 of them are duck-typed
   `*Like`/`*Root`/`*Target`/`Compiled*`/wire-parser types that exist only to satisfy recursive
   publicness of the loader option bag. Redesign the option types (real DOM types, callbacks, move
   compiler query-plan shapes to `./generated`).
5. **Fix the recursive-publicness leaks the gate cannot see, and tighten the gate to catch them**
   (`style` ABI, `server/vite`, `better-auth` options, `drizzle` `fans`, `test/harness`, `headless-ui`
   foundation types). These are unsound contracts hidden by the flat baseline.
6. **Small dead-surface removals:** `@kovojs/core#ComponentDefinition` (1, dead) and the redundant
   `@kovojs/server/app-shell/static-export` subpath (5 types already public via the root barrel).

## Definitely Remove

All four families survived adversarial verification (`confirmed`) — verified zero external consumers and
no kept signature that names them. **Three are immediate** (240 + 5 + 1 = **246 symbols / one redundant
subpath**, all clearing baseline violations now). The fourth — the **124 reducers** — is confirmed dead
to apps but **sequenced behind a prerequisite** (see its entry + Executive Summary #3): it pairs with the
`@kovoPrimitiveHandler` re-tier, which needs a `generated` tier first. So the immediate baseline drop is
~246; the reducers add ~124 once unblocked.

### `@kovojs/ui/*#*Styles` / `*Classes` / `*ClassNames` — compiled atomic-class strings (240)

**Current exposure:** public, exported from every `packages/ui/src/*.tsx` (183 `*Classes` + 46 `*Styles`

- 11 `*ClassNames`).
  **Evidence:** All 240 show `named-imports:0` AND `example-imports:0` in the inventory. They are
  StyleX-compiled atomic-class tuples (e.g. `switch.tsx:106` `switchClasses=[style.attrs(switchStyles.root).class ?? '']`).
  Grep of `examples/`, `site/`, `docs/` (excluding build caches) found **zero** imports of any of them.
  SPEC has no class-tuple export requirement; the lone "atomic classes" mention (`SPEC.md:150`) describes
  internal StyleX compile/dedupe. Component props use a separate `*StyleOverrides` type, not these consts
  (no recursive-publicness dependency).
  **Assessment:** Pure build-output implementation detail. Exporting them freezes StyleX's compiled output
  as public API — you couldn't change a style without a breaking class-string change. The `*Styles` objects
  _are_ used by the component render bodies (`switch.tsx:123`), so they stay as **module-locals**; the
  `*Classes`/`*ClassNames` consts are declaration-only in production (sole references are the package's own
  snapshot tests).
  **Recommendation:** Drop `export` from `*Styles` (keep local); delete the `*Classes`/`*ClassNames` exports.
  **Migration note:** Zero importers → non-breaking in-repo. Rewrite the package's own snapshot tests to
  read `style.attrs(localStyles.x).class`. Under the copy-in model the whole module is copied, so the
  styling capability survives as locals at zero public surface. _Single biggest baseline win: −240._

### `@kovojs/headless-ui/*#set*`/`toggle*`/`*Move`/`*Typeahead`/`*RovingIndex` — hand-authored reducers (124)

**Current exposure:** public, across all primitive subpaths.
**Evidence:** 124 functions with `named-imports:0` AND `example-imports:0` (e.g. `setSwitchChecked`/
`toggleSwitch` `switch.ts:54,79`; `accordionRovingIndex`/`accordionMoveFocus`; the `select*Adapter`
family). `examples/`/`site/` grep finds no source hits; the only consumers are the primitive files
themselves, their tests, and the non-exported primitives barrel.
**Assessment:** The reducer half of the primitive state machine that the compiler does **not** emit and
apps never import. Keeping them public freezes 124 incidental signatures (~14% of `headless-ui`
violations).
**Recommendation:** Demote to the existing `@kovojs/headless-ui/internal` subpath (already `@internal`-
tagged) or un-export.
**⚠️ Gated, not immediate.** `plans/api-cleanup-leftover.md` (closed 2026-06-19) records a standing
decision to keep this layer public "until a generated handler ABI and a different L1 island-authoring
story land," because the gallery hand-imports the reducers and emitted client modules hand-import the
sibling handlers (which therefore can't be `internal` per `rules/api-surface.md`). Sequence this with the
`@kovoPrimitiveHandler` → `generated` move (Borderline / Exec-Summary #3): build the generated tier and
move the L1 authoring path off hand-imported reducers first, then demote.
**Migration note:** Non-breaking in-repo (0 importers). Before bulk-demoting, confirm the handful of
`set*`/`toggle*` that the inventory shows as used by the kept `*Attributes`/handler functions are
relocated with their callers (they remain reachable as module-locals).

### `@kovojs/server/app-shell/static-export` — redundant public subpath (5 types)

**Current exposure:** public subpath re-exporting `StaticExportArtifact`, `StaticExportAssetArtifact`,
`StaticExportAssetInput`, `StaticExportClientModuleArtifact`, `StaticExportNonExportablePolicy`.
**Evidence:** All five = 0 imports / 0 examples. They are **already public via the root barrel**:
`StaticExportNonExportablePolicy` is re-exported directly (`index.ts:44`) and the four artifact types
are reachable through `StaticExportResult`/`StaticExportOptions` (returned/consumed by the root-exported
`exportStaticApp()`, SPEC §9.5). Only consumers are two in-repo tests, one of which (`app.test.ts:160-313`)
carries `@ts-expect-error` comments documenting that these symbols were deliberately moved to the **root**
`@kovojs/server` canonical home; the `create-kovo` starter is asserted to import from the root, not the
subpath.
**Assessment:** A second public entry point duplicating a subset of the root surface, adding a whole
extra subpath + `publishConfig` `.d.ts` target and a versioning obligation for zero capability.
**Recommendation:** Remove the subpath from `package.json` `exports` + `publishConfig` + the manifest
`apiBoundary.public`; switch the two tests to the `@kovojs/server` root specifier. All five types stay
public via the root.
**Migration note:** No external consumers; in-repo-only.

### `@kovojs/core#ComponentDefinition` — dead interface (1)

**Current exposure:** public `.` (`index.ts:87`).
**Evidence:** 0 named imports, 0 examples; repo-wide grep finds only its own declaration — no internal,
test, example, or site consumer. It duplicates `ComponentDefinitionInput` (`index.ts:110`, 25 imports /

**Migration note:** Zero importers → non-breaking.

### Recursive-publicness leaks (the gate cannot see these; flat baseline hides them)

- **`@kovojs/style/.` — `CompiledStyle`/`AtomicRule` ABI reachable from public `Style`/`StyleInput`/
  `StyleNamespaces`/`Theme`/`Vars` (5).** `engine.ts:22-54`; the helper types are exported **only** from

  cannot name the return of `style.create(...)` without reaching `@kovojs/style/internal`. **Fix:** brand
  `Style`/`Theme`/`Vars` opaque, keep the structural `AtomicRule`-bearing shape only on `/internal` for
  the compiler. Source-compatible for app authors.
- **`@kovojs/style/.` — `StyleIdentityOptions` named by public `create`/`defineVars`/`createTheme`/
  `keyframes` overloads but unexported (4).** `engine.ts:75`. The identity-bearing 2nd overload is
  compiler provenance; no app/example call passes it. **Fix:** drop the identity overloads from the public
  signatures; keep the identity variant on `/internal`.
- **`@kovojs/server/vite` — `KovoVitePlugin` return type names unexported hook-param types (1).**
  `vite.ts:38` references un-exported `KovoViteResolvedConfig`/`KovoViteHotUpdateContext`. Consumers only
  spread `kovo({app})` into `plugins[]`. **Fix:** narrow `kovo()`'s public return to a minimal
  `{name:'kovo'}`-style plugin token; keep the full hook interface internal.
- **`@kovojs/better-auth/.` — `BetterAuthCredentialMutationOptions` nameable only via `./internal`, and
  its `registry?` field names `@kovojs/server/internal/execution#MutationRegistry` (1).** Two-layer leak
  on the options of three public factories (`internal.ts:1100-1256`). Latent (examples pass inline
  `{csrf,defaultRedirectTo,guard}`). **Fix:** narrow the public options to `{csrf,defaultRedirectTo,guard,
key}`; keep `registry`/`transaction` on an `@internal` options type. Do **not** promote `MutationRegistry`
  to public to satisfy the gate.
- **`@kovojs/drizzle/.` — `fans?: readonly KovoFanAnnotation[]` element type unexported (1).**
  `drizzle-surface.ts:12,28,45`. SPEC §10.1/§11.1 (KV413 fan-out edges) grounds the feature; element type
  isn't re-exported from `runtime.ts`. **Fix:** one-line re-export of `KovoFanAnnotation`. (Borderline, not
  keep, only because of the closure gap + zero example coverage of `fans:`.)
- **⟲ `@kovojs/test/harness` — `KovoTestHarnessOptions`/`KovoTestExecOptions` reference internal types (3).**
  `harness.ts:44-49`: `touchGraph?: CoreGraph.TouchGraph` (imported from `@kovojs/core/internal/graph`),
  `verification?: DbVerificationConfig` (`@internal`), `KovoTestExecOptions = HarnessMutationOptions`
  (`@internal`). The tutorial works around it via `NonNullable<KovoTestHarnessOptions<ShopDb>['touchGraph']>`
  (`07-verification/src/app.test.ts:32`) — proof the type is otherwise unnameable. **Verifier overturned the
  "remove" reading:** these options are the SPEC §12 keystone, named by the kept `createKovoTestHarness`
  signature, with a real tutorial consumer. **Fix:** promote `core` `TouchGraph` to the public surface (it
  is already a clean `export` at `core/src/graph.ts:42`, just absent from `index.ts`); re-export
  `DbVerificationConfig`/`HarnessMutationOptions` (or inline public shapes). Keep the options public.
- **⟲ `@kovojs/headless-ui/*` — primitive types referencing lib-internal `Primitive*` foundation types
  (34).** `switch.ts:33` `SwitchPrimitiveAttributes = PrimitiveDataAttributes & …`; the foundation types
  are re-exported only via the `@internal` subpath. **Verifier softened from `remove`:** `*PrimitiveAttributes`
  are the **return type of the kept `*RootAttributes` builders** (consumed by `ui` + `examples/gallery`),

- **`@kovojs/ui` — dual distribution; reconcile both sources (1).** See Executive Summary #1. `@kovojs/ui`
  is **both** a versioned library and a copy-in starter (owner decision). The defect is that each source
  describes only one mode: `STABILITY.md:21` says copy-in only, while `public-packages.json` (`kind:library`)

- **`@kovojs/core/.` — `query()` vs `@kovojs/server#query()`: KEEP both; document the distinction (2).**

  `core#query()` as a vestigial duplicate. That was WRONG — the same "0 example imports ≠ dead" trap the

  `ComponentRegistry`; `:174-208` test `.args()`/`.refresh()` runtime behavior and the `@ts-expect-error`
  registry-key validation. Examples that bind via the server query object directly use the simpler path,
  but the handle is the SPEC-defined refresh/args binding API. **Action:** keep both; improve the JSDoc on
  each to make the declare-vs-bind layering obvious (the genuine fix for the user's "why two `query`s?").
  `core#route()` likewise stays (forced by the compiler + `href`/`Link`/`redirect`).

## Cross-Cutting Recommendations

1. **Tighten the api-surface gate to catch recursive publicness, and stop relying on the flat baseline.**
   The gate only flags `@internal`/`@generated` _tags_; it passes when a public signature names an
   _unexported-but-untagged_ helper type. This audit found that leak in `style` (`CompiledStyle`/
   `AtomicRule`, `StyleIdentityOptions`), `server/vite` (`KovoVitePlugin`), `better-auth`
   (`BetterAuthCredentialMutationOptions`→`MutationRegistry`), `drizzle` (`KovoFanAnnotation`),
   `test/harness` (`TouchGraph`/`DbVerificationConfig`/`HarnessMutationOptions`), and `headless-ui`
   (`Primitive*` foundation types). The gate should fail with the exact symbol path when a public
   signature references a non-public type.
2. **The 1,571-entry baseline conflates "dead" with "forced-public."** ~370 entries are genuinely
   removable (this report's `Definitely remove`); ~430+ are forced-public inferred return/param types that
   must instead be **documented**, not removed. Split the baseline into "to-document" vs "to-remove" so
   Phases 4-8 don't accidentally internalize forced types and re-break the gate in the other direction —
   the exact mistake the adversarial pass caught here.
3. **Use the `generated` tier consistently for compiler-emitted ABI.** `@kovojs/browser/generated` and the
   server JSX runtimes are the model; the 100 `@kovoPrimitiveHandler` functions and the `core` registry
   seeds are still parked on human-public barrels.
4. **Ban `export *` on every public barrel, not just `ui`.** `@kovojs/server` `index.ts:57-59` still uses
   it (soft, via curated sub-barrels). Convert to explicit named re-exports.
5. **Make one stability decision per ambiguous family before v1:** `@kovojs/ui` starter-vs-library
   (highest leverage); the `@kovojs/server/build` preset family's `@experimental` consistency; `core#query`
   vs `server#query`. And **document the versioned primitive deps** — `@kovojs/headless-ui` has no api-ref
   page at all, which is a v1 blocker for a package under SemVer.

## Gaps and Follow-Up

- **`@kovojs/headless-ui` machinery-type narrowing needs a real pass, not this audit's family estimate.**
  The adversarial verifier established that the ~396-count "remove" is wrong (forced-public), but the
  exact split of which machinery types are forced by a kept signature vs. genuinely demotable requires a
  per-primitive `tsc`-assisted sweep, sequenced with the reducer/handler demotions and re-running
  `scripts/api-surface-gate.mjs --write`. Treat the count as an upper bound on _review_ scope, not on
  removal.
- **`@kovojs/ui` distribution — RESOLVED 2026-06-19: dual (versioned library + copy-in starter).** This
  was the one product-owner decision the audit flagged; it is now made (Executive Summary #1). The
  remaining work is reconciliation, not a decision: `STABILITY.md` and `public-packages.json`/`package.json`
  each describe only one mode and must be updated to describe both. The library half pulls in the
  `ui` document/type/api-ref obligations; the 240 class-string un-export and the 174 Component/Props
  api-ref page are now in-scope (not contingent).
- **Method:** audit-only — no source edits. Coverage counts are reviewed-export tallies (2,137 human-public
  exports + leak-checked generated/internal subpaths). Adversarial verification covered all 7 `remove`-ranked
  findings; `borderline` findings were not independently re-verified (by definition they are judgment calls).
- **`pnpm run check:api-surface` passes** as a baseline ratchet; it is not a blocker today, which is
  precisely why the recursive-publicness leaks above are latent rather than CI-visible.
