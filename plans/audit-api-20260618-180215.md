## Definitely Remove

### `@kovojs/ui/*#headless re-export family`

**Current exposure:** `@kovojs/ui` subpaths duplicate roughly 845 `@kovojs/headless-ui` symbols through `export *`, e.g. `packages/ui/src/dialog.tsx`, `accordion.tsx`, and `command.tsx`.
**Evidence:** `STABILITY.md` says `@kovojs/ui` is shadcn-style copy-in and `@kovojs/headless-ui` is the versioned dependency; `rules/api-surface.md` bans public `export *`; `pnpm run check:api-surface` reports 872 new undocumented exports, dominated by UI primitive helper exposure.
**Assessment:** This collapses starter code into a second stable primitive namespace.
**Recommendation:** Remove the wildcard re-exports. Copied components can import headless helpers privately; generated handler ABI should import `@kovojs/headless-ui/*` directly.
**Migration note:** Replace `@kovojs/ui/<primitive>` helper imports with `@kovojs/headless-ui/<primitive>` or copied local helpers.

### `@kovojs/ui/*#*Styles/*Classes/*ClassNames`

**Current exposure:** UI-owned style/class constants are exported from component files, e.g. `buttonStyles`, `dialogClasses`, and table style constants.
**Evidence:** `STABILITY.md` defines UI as copy-in; SPEC §13.1 frames component styles as source/lowered CSS, not a stable styling API; inventory shows no meaningful example imports for these constants.
**Assessment:** These are implementation details of copied source and CSS extraction.
**Recommendation:** Stop publishing them as supported package imports; keep local to copied components if needed.
**Migration note:** Consumers that need variants should own the copied file and edit local styles.

### `@kovojs/runtime/client#loader-internal families`

**Current exposure:** `dispatchDelegatedEvent`, element state/param readers, event bus, mutation submit/apply/broadcast/queue, DOM morph, query binding/update-plan, inline query hydration, refetch, pending, and optimism rebase machinery are public from `packages/runtime/src/client.ts`.
**Evidence:** SPEC §4.4 makes loader behavior framework-owned; `rules/api-surface.md` separates generated ABI from app-facing public API; inventory shows 115 exports on `./client` with very low example demand outside starter bootstrap.
**Assessment:** Most of this is generated ABI or loader implementation, not app-authored API.
**Recommendation:** Keep only a bootstrap facade on `./client`; move generated-plan types to `./generated` and implementation to internal/private files.
**Migration note:** App client entries should call the supported install facade, not assemble loader internals.

### `@kovojs/server/app-shell/{client-modules,core,node}`

**Current exposure:** Three manifest-public subpaths with zero exports.
**Evidence:** Listed under `apiBoundary.public`; source subpaths are empty; generated docs include empty sections.
**Assessment:** Empty import paths create compatibility promises with no capability.
**Recommendation:** Remove from public boundary and package exports until a real public contract exists.
**Migration note:** None; there are no exports to import.

**Evidence:** Inventory shows 0 imports/examples; SPEC §9.5 exposes static export as request-shell replay, not artifact schema stability; root `exportStaticApp` already returns the app-facing result.
**Assessment:** Build output shape is internal implementation detail.
**Recommendation:** Move to internal/build-generated surface; keep `exportStaticApp`, `StaticExportOptions`, `StaticExportResult`, and diagnostics public.
**Migration note:** Use `exportStaticApp(...)` instead of constructing artifact records.

## Cross-Cutting Recommendations

- Treat `public-packages.json` as the gate source, but reconcile it with reality: `@kovojs/server/vite` is exported and used but not manifest-public; `@kovojs/ui` is manifest-public library while stability docs say starter/copy-in.
- Ban wildcard re-exports from every public subpath, not just root barrels. `@kovojs/ui/*` and `@kovojs/server` show why this matters.
- Add a recurring "public signature closure" check: if a public function signature names an internal, generated, or non-exported helper type, the gate should fail with the exact symbol path.
- Distinguish app-facing, tool-author, generated ABI, and test-fixture surfaces in package names/subpaths. Several current roots combine those audiences.
- For massive primitive families, document the policy once and gate mechanically: attribute factories and directly required state/support types public; reducers, geometry, parser, event-detail convenience aliases internal unless used by generated ABI.
- Resolve `@kovojs/ui` first. It will shrink the failing API gate and clarify whether remaining UI props/types should be audited as source templates or stable package API.

- `pnpm run check:api-surface` failed, so this audit cannot claim a clean API-surface gate. The failure is evidence for the UI findings.
- The skill's documented inventory path was stale for this repo (`.codex/...`); the corrected repo-local path under `.agents/...` succeeded.
- `site/gen/api` lacks pages for `@kovojs/headless-ui` and `@kovojs/ui`, so those decisions rely on manifest, source, examples, and stability/spec docs.
- Inventory/doc generation disagrees in places: inventory sees `@kovojs/server/vite`; `public-packages.json` does not classify it public.
- Some evidence is usage-by-example from in-repo apps and generated fixtures. That is weaker than external demand and should not by itself justify broad public API.
